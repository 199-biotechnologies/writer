## Key Decisions Made

1. **GPT Pro identified objective mismatch as the main bottleneck.** Not LoRA rank, not model size. The training objective (`write_chat_jsonl()` with canned prompts) teaches "answer-as-Adams" not "be-Adams". But the fix isn't just switching formats ‚Äî it's building measurement infrastructure to compare approaches.

2. **Rewrite supervision is auxiliary, not primary.** The user explicitly clarified: the product is an Adams-conditioned generator, not a rewrite system. The model should generate naturally in Adams' voice from any prompt. Rewrite-based training data is a tool for disentangling content from style, not the product architecture.

3. **Good bleed vs bad bleed** is the real quality metric. "Quite reasonably, has had to deal with the catastrophic fallout" in a bio = good (tone, framing). Arthur Dent appearing in a bio = bad (canon leakage). These must be scored separately.

4. **GPT Pro caught three inert code paths:**
   - `supports_logit_bias: false` on MLX backend ‚Üí fingerprint vocabulary bias does nothing on adapter path
   - `contrastive_enabled: true` in config ‚Üí no code reads it
   - `train_dpo()` ‚Üí returns `NotImplemented`
   These must be fixed before building new features on top.

5. **Scoring weights are misaligned with actual deficits.** AI-slop (0.15 weight) outweighs punctuation (0.10). Questions and exclamations are the biggest measured gap (8.95 target ‚Üí 0.0 actual) but are underweighted. AI-slop should be a veto, not a major scoring component.

6. **Sentence-length should be a distribution, not mean/SD.** Adams averages 6.8 words but with SD 4.3 ‚Äî he mixes fragments with flowing passages. Scoring should use histogram/EMD distance, not just compare means.

7. **`mlx_generate.py` always applies chat template** ‚Äî even if training switches to completions format, inference is still chat-wrapped. Need `prompt_mode = raw` for proper ablation.

## Gotchas & Warnings

1. **Gemma 4 26B is a thinking model.** Empty content = ran out of tokens during thinking. `max_tokens >= 4096` required. This was a bug we fixed this session ‚Äî don't regress.
2. **Use `/api/chat` not `/api/generate` for Ollama.** The old generate structs were deleted this session.
3. **mlx-lm 0.31.2 is required** for Gemma 4 support. 0.31.1 throws `Model type gemma4 not supported`.
4. **`hf_transfer` is installed** ‚Äî set `HF_HUB_ENABLE_HF_TRANSFER=1` for fast HF downloads.
5. **MLX `stream_generate` uses `sampler` not `temperature/top_p` as kwargs.** Use `make_sampler(temp=..., top_p=...)`. We hit this bug.
6. **MLX logits_processors exist** in the API (`make_logits_processors` for repetition penalty may not exist in 0.31.2 but `logits_processors` param is accepted by `stream_generate`). Need to verify exact API.
7. **The user does NOT want a rewrite system.** The product is a generative model that naturally writes in the target voice. Rewrite training data is a tool for content-style disentanglement, not the architecture.
8. **Single-agent sessions: do NOT use TaskCreate/TaskUpdate.** Global CLAUDE.md rule.
9. **User runs other processes on the machine** ‚Äî GPU sharing means training/inference may be slower than expected.
10. **The fingerprint's sentence length mean (6.8) seems low** because `unicode_segmentation::unicode_sentences()` splits on abbreviation periods.
11. **Scoring weight misalignment** ‚Äî AI-slop 0.15 > punctuation 0.10. Fix this as part of the eval harness work.
