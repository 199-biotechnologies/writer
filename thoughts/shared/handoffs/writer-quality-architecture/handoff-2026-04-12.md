1. **repetition_penalty was never applied** — `mlx_generate.py` and `mlx_worker.py` read the value but never passed it to `make_sampler()`. Fixed by implementing a custom logits processor (commit `2e12e75`) since mlx-lm 0.31.2's `make_sampler()` doesn't accept `repetition_penalty`.

3. **Silent Ollama fallback** in `write.rs` — when MLX unavailable, silently fell back to Ollama (no adapter = no voice). Now fails loudly with actionable error.
4. **"dry wit" hardcoded** in `prompts.rs` — Adams-specific text baked into all profiles' system prompts. Removed.
5. **Ablation hardcoded repetition_penalty=1.05** instead of config value. Fixed to use `cfg.decoding.repetition_penalty`.

## Key Decisions Made

1. **SimPO over standard DPO** — confirmed by research (NeurIPS 2024), Codex review, and GPT Pro. Reference-free, length-normalized, simpler. KTO rejected (needs model outputs, not just "good" corpus text).

2. **LR 5e-5 (up from 2e-5)** — literature says 2e-5 is too conservative for 26B LoRA. FlowHunt Gemma 4 fine-tune used 1e-4. Raschka found LR matters more than schedule.

3. **mask_prompt=true** — focuses learning on the writing, not canned prompts. Loss values not directly comparable to non-masked runs.

4. **Repetition penalty as logits processor** — mlx-lm's `make_sampler()` doesn't accept repetition_penalty. Implemented as a custom processor that tracks a sliding window of 100 tokens.

5. **Pair mining: best-vs-median** — BeeS (NeurIPS 2025) found medium-difficulty pairs give strongest gradient signal. Margin threshold 0.08-0.12.

6. **Resume from SFT adapter for SimPO** — don't merge first. Saves 15GB merged checkpoint. Save SimPO output to `adapters-dpo/` (never overwrite SFT adapter).

3. **eval-style is extremely slow** — uses one-shot mlx_generate.py, reloads 15GB model per generation (~80s each). Ablation uses persistent worker and is ~10x faster. Priority fix.
4. **The old adapter was trained with rank=8** (mlx-lm default) not rank=16 as configured — because we weren't passing the config YAML. The new adapter correctly uses rank=16.
